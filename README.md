## 开发流程

1. main 分支为生产分支，随时可部署，不允许直接 push。

2. 对于每一个任务，创建一个新的本地分支，命名格式为 `xyz/feature/任务名称`。其中 `xyz` 为自己姓名拼音首字母缩写，例如张三为 zs。所有提交均在该分支上完成。开发完毕后，推送该分支到远程仓库，并创建一个 Pull Request，指向 main 分支。每个人都可以直接 审核和合并自己的 PR，无需他人审批。merge 时请选择 "Squash and merge" 选项，以保持 main 分支的提交记录整洁。合并后，删除该远程分支，注意不要删错了。

3. 经常性地 fetch 和 pull main 分支的最新代码，保持本地 main 分支与远程同步。每次在创建新分支前，确保本地 main 分支是最新的。
